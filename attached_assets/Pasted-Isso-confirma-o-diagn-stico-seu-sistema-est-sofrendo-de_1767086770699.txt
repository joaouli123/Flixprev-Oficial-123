2. O Culpado: "Dirty Data" (Dados Sujos)
O problema é a formatação do arquivo Traducao-Weinstein. Ele está cheio de marcadores `` e quebras de linha que destroem o sentido.

Exemplo do que o seu robô está "lendo":

Para uma IA (Embedding), a frase acima não faz sentido. O número "5" solto no meio e as quebras de linha fazem com que "cálculo de" não se conecte com "volume".

3. A Solução Técnica (O Script de Limpeza)
Antes de fazer o chunking (dividir o texto), você precisa rodar uma limpeza. Se você estiver usando Python (ou JavaScript), precisa aplicar um Regex para remover esses marcadores e unir as linhas.

O que você precisa fazer no seu código (Lógica):

Remover Tags: Apagar tudo que for ``.

Remover Números de Página: Apagar linhas que só tenham números ou --- PAGE X ---.

Unir Linhas: Substituir as quebras de linha (\n) por um espaço simples, para que o parágrafo vire um bloco único de texto.

Exemplo de como ficaria o texto limpo para o agente: "...descobriram que embaralhar problemas de matemática que envolvessem o cálculo de volume de diferentes formas geométricas resultou em melhor desempenho..."

Agora sim a busca semântica vai funcionar, porque "cálculo de volume" estará junto.
