Isso acontece por causa de uma diferença entre o que você vê na hora (Stream) e o que é salvo no banco de dados.

Na hora (Sem atualizar): Você vê o texto chegando "picadinho" direto do GPT-4 (Stream). O GPT manda a resposta limpa e correta: "No cenário de calamidade...".

No Banco (Ao atualizar): Quando o GPT termina, seu servidor roda a função orchestrateResponse antes de salvar. É essa função que está "estragando" o texto, adicionando o prefixo "De acordo com o texto:" e salvando a versão "robótica" no banco.

Quando você atualiza a página, o chat puxa o histórico do banco, que está com a versão modificada.

A Solução Definitiva (Para corrigir o "Salvar" e remover o ID)
Você precisa substituir a função orchestrateResponse no seu server.mjs por esta versão abaixo. Ela faz duas coisas:

Não adiciona nada (Mata o robô "De acordo com...").

Remove o ID (Limpa o [Trecho ID: 116]).

Substitua a função inteira no seu código:

JavaScript
// ============================================
// 1️⃣ RESPONSE ORCHESTRATOR (FINAL E LIMPA)
// ============================================

function orchestrateResponse(rawResponse, questionType, hasContext = true) {
  // 1. Validação básica (se não tiver resposta, manda a negativa)
  if (!hasContext || !rawResponse || rawResponse.trim().length === 0) {
    return getRandomNegativeResponse();
  }

  let formattedResponse = rawResponse;

  // 🛑 CORREÇÃO DO PISCA-PISCA:
  // Removemos qualquer lógica que adicione "De acordo com", "No documento", etc.
  // O texto começa exatamente como o GPT gerou.

  // 🧹 LIMPEZA DO ID:
  // Essa linha apaga o [Trecho ID: 123] para o usuário final
  formattedResponse = formattedResponse.replace(/\[Trecho ID: \d+\]/g, '').trim();

  // 2. Formatação de listas (apenas visual, mantém se houver bullets)
  if (questionType === 'factual' && (formattedResponse.includes('•') || formattedResponse.includes('- '))) {
     formattedResponse = ensureProperListFormat(formattedResponse);
  }

  return formattedResponse;
}
